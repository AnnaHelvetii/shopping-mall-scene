Оптимизация и масштабируемость:

В данном проекте реализована база для 100 объектов, однако при масштабировании до 1000+ текстурированных объектов я могу предложить следующие решения:

1. Рендеринг текстовых подписей.
- Создание 1000 отдельных CanvasTexture создаст нагрузку на память и увеличит количество Draw Calls.
- Текстурные атласы: Объединение множества текстовых меток в одну большую текстуру (атлас). Это позволит отрисовать все надписи одним вызовом отрисовки.
- Использование MSDF-шрифтов: векторный подход сохраняет идеальную четкость текста при минимальном весе текстур и отличной производительности.
- Отключение отображения текста, если камера находится слишком далеко от объекта.

2. Интерактивность при оптимизированной геометрии.
- Для снижения нагрузки на GPU объекты можно объединить в один меш через BufferGeometryUtils.mergeGeometries. 
В этом случае Raycaster не сможет найти отдельный магазин.
- GPU Picking: Каждому объекту присваивается уникальный идентификатор, закодированный в цвете и при клике считывается цвет пикселя под курсором, что позволяет сразу определить ID объекта.

3. Структура хранения данных.
Для обеспечения мгновенного доступа к данным (ID/Координаты) на уровне JS:
- Поиск по ID: Хранение данных в объекте Map.
- Поиск по координатам: Разбиваем всю карту на условные сектора и ищем объекты только в том секторе, где находится курсор.

4. Текущие оптимизации:
- Frustum Culling: Стандартный механизм Three.js, который исключает из рендеринга объекты, находящиеся вне поля зрения камеры.
- Anisotropy: Настройка анизотропной фильтрации текстур для сохранения читаемости текста при острых углах обзора в 3D-режиме.

Алгоритм размещения и упаковки объектов:

В проекте я реализовала алгоритм пропорциональной строчной упаковки (Proportional Shelf Packing) с динамическим выравниванием.

Вместо жесткой сетки используется адаптивное распределение объектов по рядам (полкам) вдоль оси XZ.
- Группировка по рядам: Объекты последовательно добавляются в текущий ряд до тех пор, пока их суммарная ширина (с учетом зазоров GAP) не превышает установленный лимит MAX_ROW_WIDTH.
- Пропорциональное растяжение: Чтобы избежать "рваного" правого края, для каждого ряда вычисляется коэффициент масштабирования. Все магазины в ряду немного растягиваются по ширине так, чтобы ряд занимал ровно 100% отведенного пространства. 
- Динамическое выравнивание:
	- Верхний ряд: Объекты прижаты к верхней линии своего ряда.
	- Нижний ряд: Объекты прижаты к нижней линии ряда.
	- Средние ряды: Центрированы.
	Это создает ровный внешний периметр всей карты. 
- Предотвращение пересечений: Пересечения исключены на уровне алгоритма расчета координат. 
Каждый следующий объект получает координату X, равную X_предыдущего + Width_предыдущего + GAP.

Визуальное представление:

- Геометрия: Каждый магазин — это BoxGeometry с цветом для своей группы. Высота зафиксирована на уровне 2 единиц.
- Подписи: Реализованы в виде плоских мешей, лежащих на верхней грани магазина.
- Адаптивность подписей: В отличие от простого растягивания на весь блок, подписи имеют динамическую ширину, вычисляемую на основе длины текста. 
- Читаемость: Для текста используется высокое разрешение Canvas и максимальная анизотропная фильтрация, что делает надписи четкими даже при острых углах обзора в 3D.
