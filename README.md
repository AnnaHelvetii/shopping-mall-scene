# Архитектура проекта

Проект построен с использованием модульной структуры для удобства масштабирования и поддержки.

- src/scene/ — инициализация сцены, освещения, рендерера и создание объектов;
- src/cameras/ — конфигурация камер (Perspective) и параметров переключения видов;
- src/interaction/ — обработка событий мыши (Hover, Click) и управление камерой (OrbitControls);
- src/data/ — массив данных магазинов;
- src/utils/ — вспомогательные скрипты (обработка ресайза);
- script.js — точка входа, связывающая все модули и запускающая Render Loop;

# Инструкция по запуску

Для локального запуска проекта необходим локальный сервер:

- Клонируйте репозиторий: git clone https://github.com/AnnaHelvetii/shopping-mall-scene
- Запустите локальный сервер. Если у вас установлен VS Code, используйте расширение Live Server. Или через Node.js: npx serve.
- Откройте браузер: Перейдите по адресу http://localhost:5500 (или другому порту).

# Алгоритм размещения объектов

Для размещения 100 объектов без пересечений реализован алгоритм пропорциональной строчной упаковки:
- Распределение: Магазины объединяются в ряды. Как только сумма ширин объектов превышает лимит ряда, создается новый ряд.
- Flex-растяжение: Объекты в ряду пропорционально увеличивают свою ширину, чтобы заполнить ряд целиком. Это создает ровные края по бокам и устраняет пустоты.
- Выравнивание: Для имитации коридоров ТЦ верхний ряд выровнен по верхней границе, а нижний — по нижней.
- Координаты: Позиция каждого объекта рассчитывается на лету с учетом его габаритов и заданного отступа (GAP), что математически исключает наложение.

# Архитектура и оптимизация

1. Как оптимизировать рендеринг 1000+ текстовых подписей?
   - Использовать Текстурные атласы (Texture Atlas) для объединения всех надписей в одну текстуру, снижая Draw Calls.
   - Применять MSDF-шрифты (Multi-channel Signed Distance Fields) для сохранения четкости при малом весе текстур.
   - Внедрить LOD (Level of Detail): скрывать текст, если камера находится далеко.
2. Как обрабатывать клики при объединении геометрии?
   - Использовать GPU Picking: отрисовывать объекты в невидимый буфер с уникальными цветами-идентификаторами.
   - Хранить упрощенные невидимые меши специально для Raycaster, пока визуально отображается один сложный меш.
3. Структура данных для быстрого поиска:
   - По ID: Использовать Map (JS object/map).
   - По координатам: Использовать Spatial Hashing (разбиение сцены на сектора) или Quadtree, чтобы проверять только те объекты, которые находятся в зоне клика.
